## 6. 技术兼容性与多端支持
- **统一 API 层**：后台通过 RESTful/JSON API 对外提供数据，小程序与 Web 端共用同一套接口，仅需在前端根据终端特性调整 UI 与交互。
- **鉴权方案**：建议采用 Token（如 JWT）或 Session+Token 机制。小程序端通过 `wx.login` 获取后端 Token，Web 端可通过账号密码/手机号登录，后端统一校验，保证权限逻辑一致。
- **域名与安全要求**：小程序端必须使用备案的 HTTPS 域名（443 端口）。如果 Web 端也部署同域，可直接共享；若多域部署需在后端/网关层启用 CORS 控制。
- **静态资源管理**：首页 Banner、商品图片等推荐位素材统一存储于对象存储或 CDN，接口返回绝对 URL，前端（小程序/Web）均可直接展示。
- **规格/业务字典**：尺寸、材质、风格等字典数据由后台统一维护，接口返回结构化字段，前端无需写死值，避免多端不一致。
- **版本兼容策略**：新增字段采用向后兼容方式（前端未使用时忽略即可）；如需废弃字段，先在文档中声明并提供过渡期，便于多端同步调整。
- **登录跳转差异**：小程序与 Web 登录入口不同，但登录完成后均调用同一套业务接口。可在接口层通过 `User-Agent` 或参数区分终端，用于埋点统计或灰度策略。
